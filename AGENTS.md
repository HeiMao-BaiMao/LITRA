# Project-Specific Agent Instructions

## ツールの積極的な活用

コードの探索・理解・変更を行う際は、以下のツールを積極的に使用してください。

- **ccc（CocoIndex Code）**: セマンティックなコード検索やインデックス管理に使用します。
  - 概念や機能に関連するコードを探す場合は `ccc search <query>` を優先して使ってください。
  - インデックスが古くなっている可能性がある場合は `ccc index` または `ccc search --refresh` で更新してください。
  - `ccc` が未初期化でエラーになった場合は、プロジェクトルートで `ccc init` してから `ccc index` を実行し、元の作業を再試行してください。
- **Serena（MCP ツールセット）**: シンボル検索、定義・参照の探索、診断情報の取得、シンボル単位の編集などに使用します。
  - 特定の関数・クラス・モジュールを探す場合は `mcp__serena__find_symbol` などを優先的に使ってください。
  - 影響範囲を調べる場合は `mcp__serena__find_referencing_symbols` や `mcp__serena__find_implementations` を活用してください。
  - ファイル全体を書き換える必要がない限り、`mcp__serena__replace_symbol_body` などシンボル単位の編集を優先してください。

## ccc の探索対象制限に関する許可

ccc の探索対象（含める / 除外するパス、言語設定、その他の設定ファイル）については、作業に最適な範囲になるよう**自由に編集・調整して構いません**。ユーザーへの追加確認は不要です。

ただし、以下に注意してください。

- `.env`、資格情報、API キー、トークンなどの機密情報が含まれるファイルは除外対象とし、検索・インデックスに含めないようにしてください。
- ビルド成果物（`dist/`、`target/`、`node_modules/` など）が誤って含まれていないか確認し、必要に応じて除外してください。
